# 专业整理设计

## 目标与范围

将现有“浏览/筛选”原地升级为“专业整理”，保留全部现有浏览、扩展名筛选、勾选、统计、缩放和外部看图能力，并新增非破坏式页面排序与编辑。所有操作只影响导出的 PDF，不修改原始图片。

第一阶段仅支持文件夹输入。压缩包继续使用现有一键导出流程。

本阶段包含：

- 拖拽调整页面顺序
- 左旋、右旋、裁边、设为封面
- 单页、从左到右、从右到左三种阅读方向
- 页面视图与 PDF 输出布局预览

本阶段不包含：

- 独立“排除”命令；继续使用现有勾选框
- 空白页、重复页和方向异常检查
- 横向跨页识别与拆分
- 压缩包专业整理

## 界面与交互

主窗体的“浏览/筛选”按钮改名为“专业整理”。专业整理窗口保留现有左侧扩展名、选择统计和缩放卡片，在左侧剩余区域新增“输出预览”卡片：

```text
输出预览
[页面视图] [PDF预览]

阅读方向
○ 单页
○ 左→右
○ 右→左
```

页面视图沿用当前缩略图网格。用户可以拖动一张缩略图，将其插入目标缩略图之前；扩展名筛选隐藏的页面保持原相对顺序。PDF 预览切换为暗色成品阅读器，不允许拖拽。纵向模式使用类似浏览器的连续像素滚动，页面按可用宽度等比放大，左右边距和页间黑边均约 12px，让图像区域尽量占满；页面上下排列，滚动到页边界时允许同时看到上一页底部和下一页顶部。纵向始终从上到下，不受阅读方向影响。横向模式每个视口只显示一张最终输出页，滚轮和滚动条按整页切换；左→右时从左开始、右→左时从右开始，并可点击图片两侧的黑色留白翻页。两种模式都不使用超大原图画布，只绘制当前可见页并预加载相邻页。“预览滚动”和“阅读方向”是两个独立选择组，默认纵向。单页模式每张图对应一页；双页模式将相邻竖页合成为一张成品页，封面始终单独成页，横图沿用 Core 当前的单页规则。预览仅缓存当前可视页及邻近页的中等分辨率合成图。

右键某张缩略图显示：

```text
左旋 90°
右旋 90°
裁边…
设为封面
恢复此页
```

操作只作用于右键命中的图片。“设为封面”取消此前的封面标记、将该页移到导出顺序首位，并在缩略图上显示封面标记。用户确认专业整理后，主窗口预览同步显示已选封面；没有封面标记时显示首个已选页面。“恢复此页”清除该页的旋转、裁剪和封面，并将其放回文件夹自然排序中的原始位置，不改变该页的勾选状态。

所有页面视图和 PDF 预览缩略图都显示非破坏式操作标记：

```text
[封面] [↺ 90°] [已裁边]
```

只显示实际存在的标记。存在任一操作的缩略图使用橙色状态点，便于滚动浏览时快速定位；右键菜单中的“恢复此页”仅在该页有改动时启用。

左侧现有总页面/选择统计卡片增加始终可见的“一键全部重置”按钮。点击后先确认，确认后清除全部旋转、裁剪和封面，并恢复文件夹自然排序；保留阅读方向、现有勾选、扩展名筛选与缩放状态。拖拽后的页面不显示“已移动”标记，用户通过文件名 ID 判断顺序。第一阶段不实现多步撤销/重做历史。

裁边打开模态单图窗口。窗口按当前旋转方向显示图片，裁剪框外使用暗色遮罩，四边和四角均可拖动，底部提供“重置、取消、应用”。裁剪区域保存为相对当前旋转画面的 0–1 比例坐标。后续旋转同时换算裁剪坐标，使可见区域保持一致。

点击“确定”后，页面顺序、勾选状态、编辑参数和阅读方向回传主窗体；阅读方向同步主窗体已有的输出格式选项。点击“取消”不改变主窗体状态。

## 数据模型与传递

每个页面仅保存：

- 原始文件路径
- 文件夹自然排序中的原始位置
- 旋转角度：0、90、180、270
- 裁剪区域：Left、Top、Width、Height，均为 0–1 比例
- 是否选中
- 是否封面

`GalleryForm` 使用页面列表的实际顺序作为导出顺序。为了避免当前文件清单在 Lua 和 Core 中被再次排序，编辑后的导出改用带版本头的临时页面清单；每行包含 Base64 编码路径和页面参数。Base64 避免路径中的空格、制表符和非 ASCII 字符破坏格式。正常文件夹导出继续沿用现有排序逻辑。

主窗体启动 Core 时通过新的 `--page-manifest` 参数传入清单，并在进程结束后删除临时文件。Core 校验版本、字段数量、旋转角度和裁剪范围；无效清单输出明确错误并停止，不静默回退到原始目录导出。

## Core 处理

Core 按清单顺序逐页加载图片，先旋转，再按比例裁剪，然后交给现有 PDF 写入流程。每次只保留当前页；双页模式最多额外保留一张待合并页。页面写入后立即释放位图。

PDF 预览与 Core 使用同一套页面分组规则：

- 封面总是单页
- 单页模式不组合
- 双页模式只组合相邻竖页
- 横图保持单页，并中断当前组合
- 末尾单张页面保持单页

这样预览顺序与最终 PDF 一致，不生成临时 PDF，也不缓存全尺寸文档。

## 缓存与资源

缩略图继续按需加载，但增加固定数量的最近使用上限；超出上限时释放最久未显示的缩略图。裁剪窗口只在打开期间持有一张全尺寸位图，关闭时立即释放。PDF 预览复用页面缩略图，不生成第二套完整缓存。

## 错误处理

- 源图片在整理后被删除或移动：导出前报告具体文件并停止。
- 图片无法解码：沿用现有跳过行为，但在完成信息中报告跳过数量。
- 裁剪框为空或越界：禁止应用。
- 临时清单解析失败：Core 写入标准错误并停止。
- 临时清单删除失败：不影响已完成的 PDF，但保留可观察的错误信息。

## 验收与验证

- 原浏览/筛选的扩展名、全选、全不选、统计、缩放、重置和双击看图功能无回归。
- 拖动页面后，页面视图、PDF 预览和实际 PDF 顺序一致。
- 左旋、右旋和裁边在缩略图、PDF 预览和实际 PDF 中一致。
- 被旋转、裁边或设为封面的页面显示对应标记；未编辑页面不显示标记，拖拽排序不显示标记。
- “恢复此页”只恢复命中页面的操作且保留勾选；左侧“一键全部重置”清除全部页面操作并恢复自然排序，同时保留阅读方向、筛选、勾选和缩放。
- 设为封面后该页位于首位，双页模式下独占第一页。
- 三种阅读方向与主窗体选项同步，双页左右顺序正确。
- 取消专业整理不会改变主窗体已有选择。
- 数千页目录不会因缩略图数量持续增加内存占用。
- 含中文、空格和特殊字符的路径能够通过页面清单导出。
